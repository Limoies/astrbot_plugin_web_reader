## 🐛 踩过的 13 个坑

### 代码逻辑
1. **正则吞掉URL** — `--\S+(\s+\S+)?` 正则把URL当参数吃了
   → ✅ 改用 `split` + 过滤 `--` 前缀
2. **LLM端点拼接** — 字符串硬拼容易出错
   → ✅ 用 `urljoin` 更安全
3. **错误信息泄露API Key** — 直接把响应体返回给用户
   → ✅ 只返回状态码

### AstrBot 兼容性
4. **`@filter.on_message()` 不存在** — 没查文档就用了不存在的API
   → ✅ 先确认官方文档支持的装饰器列表
5. **缩进错误** — 装饰器比函数多缩进了4格，导致IndentationError
   → ✅ 写完代码做 `py_compile.compile()` 语法检查
6. **导入路径不统一** — `astrbot.api.platform` vs `astrbot.core.platform`
   → ✅ 以官方示例插件为准
7. **缺少生命周期方法** — `initialize()` / `terminate()` 没有实现
   → ✅ 好的实践是都加上

### 环境与部署
8. **`\\?\` 长路径前缀** — Node.js 无法处理 Windows 长路径
   → ✅ 运行时用 `try` 包裹，动态修复路径
9. **依赖装错Python环境** — pip install 装到了系统Python，不是AstrBot的
   → ✅ 用 `AstrBot\backend\python\python.exe -m pip install`
10. **`/` 被聊天框吃掉** — 命令前缀斜杠没了导致URL解析失败
    → ✅ 参数解析代码加上无斜杠前缀作为兜底

### 功能设计
11. **`_is_safe_url` 定义了没调用** — SSRF防护形同虚设
    → ✅ 写完功能后检查调用链，确认被使用
12. **CDP模式失败回退到空URL** — 静态模式用空URL请求抛异常
    → ✅ CDP模式失败直接报错，不回退到静态模式
13. **纯文本压成一长串** — `--no-llm` 模式所有内容粘在一起
    → ✅ 后处理加段落格式化（句号换行、编号分段）

## 💡 经验法则

### 开发前
- [ ] 确认目标框架的API文档
- [ ] 参考官方示例项目的代码结构
- [ ] 确定Python环境路径

### 编码时
- [ ] 写完一个功能马上 `py_compile` 检查语法
- [ ] 每改完一个bug就 `git commit`
- [ ] 关键路径加日志，但注意不要泄露敏感信息
- [ ] 定义了的函数确认是否被调用

### 调试时
- [ ] 不要在插件代码中频繁print → 用logger
- [ ] 错误信息要给足上下文（如显示消息原文）
- [ ] 先本地模拟测试，再部署到AstrBot测试

### 开源前
- [ ] 检查有无硬编码的用户名/路径
- [ ] `.gitignore` 提前配置好（排除 `.vs/`、`__pycache__`）
- [ ] README写清楚依赖和配置说明
- [ ] 添加 LICENSE 文件
